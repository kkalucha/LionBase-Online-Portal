Choosing Predictors - Theory¶

Bias-Variance Tradeoff - Refresher¶

In the Linear Regression - Theory notebook, we introduced the Bias-Variance Tradeoff. As bias goes up, variance goes down and vice versa. We can interpret bias and variance as the following:

  • Bias: A model's ability to predict the response variable in a training dataset
  • Variance: A model's stability of predictions across different training datasets

When there are many predictor variables used in linear regression, there is a greater chance to overfit (less bias, more variance), which results in poor predictions.

How can we reduce variance without increasing bias significantly? Two options:

  • Subset selection
  • Shrinkage

In subset selection, we try out different combinations of $t$ predictors and see which works best.

In shrinkage, we add in penalty terms that constrain the $\beta$ values or eliminates certain coefficients.

Subset Selection¶

In subset selection, we try out different combinations of $t$ predictors and see which works best.

There are two main methods:

  • Best Subset Selection
  • Forward Stepwise Selection

Best Subset Selection is more thorough, but forward stepwise selection is more efficient.

Best Subset Selection¶

Here, we basically try out every possible combination of $t$ predictor variables and see which has the best performance, where performance is defined by minimizing $MSE$. For example, suppose we have three predictor variables, $\{A,B,C\}$, then the combinations we would try are:

  • A
  • AB
  • ABC
  • B
  • BC
  • C

This is very expensive, so this is only OK when $t$ is relatively small.

Forward Stepwise Selection¶

When $t$ is large, it is more efficient to use forward stepwise selection.

  • Add predictors to the model, one-at-a-time
  • Add the variable that gives the best improvement when added to the model
  • Repeat until all $p$ features have been added
  • Choose the best model out of all $p+1$ models (it is p+1 because we start with zero predictors, which is just $\beta_0$, or the null model)

For example, suppose we have three predictor variables, $\{A,B,C\}$, we might end up doing something like:

  • Step 1: Try out just A, just B, and just C. B provides the best performance. The model's predictors are now the set {B}
  • Step 2: Try out BA, and BC. BC provides the greatest increase in performance. The model's predictors are now the set {B,C}.
  • Step 3: Try out BCA. This is no better than {B,C}.
  • So, we settle on a model with predictors {B,C}

Shrinkage¶

In shrinkage, we aim to constrain the number of predictors or to shrink the $\hat{\beta}$ values in a selective manner to reduce variance while keeping the increase in bias negligible. There are three other variations of linear regression that aim to do this by adding in shrinkage penalty terms. This is also known as regularization.

  • Ridge Regression (shrinks the $\hat{\beta}$)
  • Lasso Regression (removes uselsss predictors)
  • Elastic Net Regression (mix between Ridge and Lasso)

There is no proof that any of these forms of regression above are more or less valuable than another. The only way to know is through model selection and assessment (found in a different section).

However, all three methods are useful for building a model.

Ridge Regression¶

$min_{\beta_0,\beta}\{\sum_{i=1}^{n}(y_i-\beta_0-\beta'x_i)^2 + \color{red}{\lambda\sum_{j=1}^{p}\beta_j^2}\}$, where $\lambda\geq0$ is a tuning parameter

The red term is based on the $L_2$ norm and is the shrinkage penalty. It penalizes how large the $\beta$ values are.

  • As $\lambda$ increases, the variance decreases but the bias increases because the flexibility of the model decreases
  • $\lambda=0$ is equivalent to the least squares estimate

In particular, ridge regression shrinks the coefficients of correlated predictors.

Lasso Regression¶

$min_{\beta_0,\beta}\{\sum_{i=1}^{n}(y_i-\beta_0-\beta'x_i)^2+\color{red}{\lambda\sum_{j=1}^{p}|\beta_j|}\}$, where $\lambda\geq0$ is a tuning parameter

The red term is based on the the $L_1$ norm and is the shrinkage penalty.

  • As $\lambda$ increases, the variance decreases but the bias increases because the flexibility of the model decreases
  • $\lambda=0$ is equivalent to the least squares estimate

Because lasso uses $\lambda\sum_{j=1}^{p}|\beta_j|$ as its regularization term, some coefficients will be 0 exactly.

  • This leads to a reduction in the number of predictor variables

Elastic Net Selection¶

$min_{\beta_0,\beta}\{\sum_{i=1}^{n}(y_i-\beta_0-\beta'x_i)^2+\color{red}{\lambda\sum_{j=1}^{p}((1-\alpha)\beta_j^2+\alpha|\beta_j|)}\}$

This combines both penalty terms (the $L_1$ and $L_2$ norms) to select variables like in lasso regression but shrink the coefficients of correlated predictors like in ridge regression.
