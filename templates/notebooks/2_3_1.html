Logistic Regression Theory¶

Classification¶

Classification applies to cases where we are trying to predict discrete outcomes that might not be numeric. Examples include:

  • Whether a person purchases a product or not
  • Who an individual will vote for
  • What outcome a patient has following a surgery

In cases where we are considering only two possible outcomes, logistic regression is very useful.

  • Yes, it is a regression method! We will go into how it (and any regression method) can be turned into a classification tool

Logistic Regression Introduction¶

Logistic regression, also known as logit regression, is a tool for modeling the probability of a certain class existing (e.x. yes or no).

  • Binomial Logistic Regression: Applies to one class with two outcomes
  • Multinomial Regression: Applies to one class with more than two outcomes

We will start with binomial logistic regression, which can be shortened to just saying logistic regression (yes, another confusing part of the wording). In other words, if someone is discussing logistic regression, assume it is binomial unless otherwise specified!

Logistic regression models (also known as a logit model) the probability between 0 and 1.

  • To turn a logit model into a classification tool, set probability thresholds. E.x. "If the output is greater than or equal to 0.6, return YES, else return NO"
  • It is most common for the threshold to be set at 0.5, but it can vary
  • Linear regression is not appropriate for this as the values of a linear regression can extend outside the range $[0,1]$

The key idea is to model the probability of an outcome and define at what thresholds the outcome is decided.

Binomial Logistic Regression¶

Whenever we think of a class with two outcomes, we view success as "1" and failure as "0":

  • E.x. If a person purchases X, mark as 1, else mark as 0

Logistic regression estimates the log-odds of success (i.e. $q(X)=Pr(Y=1|X)$) based on a linear combination of the predictors:

  • $ln(\frac{q(X)}{1-q(X)}) = \beta_0 + \beta_1X_1 + \beta_2X_2 + ... + \beta_pX_p$

This can also be written as:

  • $q(X)=Pr(Y=1|X) = \frac{exp(\beta_0 + \beta_1X_1 + \beta_2X_2 + ... + \beta_pX_p)}{1 + exp(\beta_0 + \beta_1X_1 + \beta_2X_2 + ... + \beta_pX_p)}$

The notation $Pr(Y=1|X)$ can be read as the "probability of Y = 1 (i.e. success) given X."

  • Y is the dependent variable
  • X describes the independent variables

Another way to view this is we are applying the sigmoid function $p = \frac{1}{(1+e^{-y})}$ on the linear regression equation.

Maximum Likelihood Estimation and Deviance¶

The $\beta$ values are estimated by selecting the $\hat{\beta_0}, \hat{\beta_1}, ..., \hat{\beta_p}$ values that maximize the likelihood function:

  • $\prod_{i=1}^{n} f(y_i|x_i)$

This is basically the cumulative product of each outcome $y_i$ given $x_i$ and can be rewritten as:

  • $\prod_{i=1}^{n}(q(x_i))^{y_i}(1-q(x_i))^{1-y_i}$

which is equivalent to:

  • = $\prod_{i=1}^{n} [\frac{exp(\beta_0+\beta^Tx_i)}{1+exp(\beta_0+\beta^Tx_i)}]^{y_i} \cdot [\frac{1}{1+exp(\beta_0+\beta^Tx_i)}]^{1-y_i}$

In case you are unfamiliar with the notation:

  • $\beta_0$ refers to the intercept as before
  • $\beta$ refers to $\beta_1, ..., \beta_p$ -- this is a vector of $\beta$ values
  • $\beta^T$ refers to taking the transpose of the $\beta$ values, so that when it is multipled by $x_i$ the dimensions work out correctly
  • It is good to know, but OK if you are not entirely familiar with this as it dives more into linear algebra

Why do we use product operator ($\prod$) instead of a summation ($\sum$)?

  • An intuitive way to view this is that likelihood is the same as probability, which is between 0 and 1
  • The maximum probability that can exist is 1 (by definition)
  • Suppose we have three rows in our dataset, each with probability 1
  • The maximum likelihood estimation (MLE) should be $1\cdot1\cdot1=1$ not $1+1+1=3$

Maximizing the likelihood function is equivalent ot minimizing the deviance, which is the negative log of the likelihood:

  • $D=-[sum_{i=1}^{n}y_ilog(q(x_i))+sum_{i=1}^{n}(1-y_i)log(1-q(x_i))]$

where $q(x_i)=\frac{exp(\hat{\beta_0} + \hat{\beta_1x_{i1}}+...+\hat{\beta_px_{ip}})}{(1+exp(\hat{\beta_0} + \hat{\beta_1x_{i1}}+...+\hat{\beta_px_{ip}}))}$

We usually focus on minimizing deviance as it is a significantly less expensive question to solve. The real significance of taking the negative log of the likelihood is it turns the equation into a convex optimization problem, which has much better time efficiency. This is covered more in-depth in the OPERATIONS RESEARCH module.

Remember that Logistic Regression provides a probability of success.

  • $\hat{q}(x) = \frac{exp(\hat{\beta_0} + \hat{\beta_1x_{i1}}+...+\hat{\beta_px_{ip}})}{(1+exp(\hat{\beta_0} + \hat{\beta_1x_{i1}}+...+\hat{\beta_px_{ip}}))}$

where $\hat{\beta_0}, \hat{\beta_1}, ..., \hat{\beta_p}$

How can we use $\hat{q}(x)$ to classify whether or not someone passed the exam?

  • Set a threshold $\tau$ where if $\hat{q}(x)\geq\tau$, then label as a success, else a failure
  • $\tau$ may be greater than or less than 0.5
  • Selecting $\tau$ should be based on prediction metrics at different $\tau$ thresholds (e.x. what is the misclassification rate at $\tau = 0.5$ as opposed to $\tau = 0.2$?) -- reference Model Selection and Assessment to learn more about this

Linear Regression as Classification¶

  • The same threshold strategy can be applied to linear regression for classification purposes, though it usually is not as effective as logistic regression
  • Why use linear regression?
    • Linear regression is more interpretable than logistic regression

Multinomial Logistic Regression¶

Assume we have $K\geq2$ possible (unordered) outcomes $\{0,1,2,...,K-1\}$

  • We go up to $K-1$ instead of up to $K$ because the absence of all other outcomes implies the last outcome $K$
  • E.x. Suppose there are three outcomes: Yes, No, Maybe. We can define this as $Yes = (1,0)$, $No = (0,1)$ and $Maybe = (0,0)$.

We can describe the probability of each outcome $k = 1,...,K-1$ with:

  • $q_k(x)=Pr(Y=k|X)=\frac{exp(\beta^{(k)\top}X)}{1+\sum_{l=1}^{K}exp(\beta^{\top}X)}$

The $\top$ symbol indicates to take the largest element in the lattice. For each option $k$ the coefficients $\beta^{(k)}$ capture the importance of the covariates for option $k$.

For option 0:

  • $q_0(x)=Pr(Y=k|X)=\frac{1}{1+\sum_{l=1}^{K}exp(\beta^{\top}X)}$

Option 0 is generally the default and serves as a normalization (i.e. $\beta^{(0)} = 0$)

Shrinkage¶

Recall we are trying to minimize deviance:

  • $D=-[sum_{i=1}^{n}y_ilog(q(x_i))+sum_{i=1}^{n}(1-y_i)log(1-q(x_i))]$

where $q(x_i)=\frac{exp(\hat{\beta_0} + \hat{\beta_1x_{i1}}+...+\hat{\beta_px_{ip}})}{(1+exp(\hat{\beta_0} + \hat{\beta_1x_{i1}}+...+\hat{\beta_px_{ip}}))}$

To apply shrinkage penalties on $\beta$, just add the penalties to the deviance $D$.

  • Ridge: $D + \lambda\sum_{j=1}^{p}\hat\beta_j^2$
  • Lasso: $D + \lambda\sum_{j=1}^{p}\hat|\beta_j|$
  • Elastic: $D + \lambda\sum_{j=1}^{p}((1-\alpha)\beta_j^2+\alpha|\beta_j|)$
